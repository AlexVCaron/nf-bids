== End-User Installation (Nextflow Plugin Registry)

The plugin is published to the https://registry.nextflow.io/plugins/nf-bids[Nextflow Plugin Registry].
No build tools are required; Nextflow downloads and caches the plugin automatically on first run.

[WARNING]
.MacOS users
====
MacOS ships an outdated version of `BASH`. To use `nf-bids`, update it using `Homebrew` :

[.wrap,console]
----
brew update
brew install bash
----
====

Add the following block to your `nextflow.config`:

[source,groovy]
----
plugins {
    id 'nf-bids@{plugin-version}'
}
----

Then import the `Channel.fromBIDS` factory in your workflow:

[source,groovy]
----
include { fromBIDS }                      from 'plugin/nf-bids'
----

*What this gives you:* Nextflow resolves and caches the plugin JAR under
`~/.nextflow/plugins/nf-bids-{plugin-version}/` the first time the workflow runs.
*Failure mode:* If the registry is unreachable or the version string is wrong,
Nextflow aborts before execution starts with a plugin-resolution error.
Re-check the version string against the
https://registry.nextflow.io/plugins/nf-bids[registry listing].

For plugin development and local source installation, use
xref:development:guides/building.adoc[Development Building & Installation].
